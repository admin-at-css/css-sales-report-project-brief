## ğŸ“‹ Problem Statement

### Workflow Gap Yang Ditemukan

**Current Documented Flow:**
```
Step 1: User membuat Company (separate screen)
Step 2: User membuat Contact (separate screen)
Step 3: User membuat Project (separate screen)
Step 4: User membuat Report (select project dari dropdown)
```

**Real-World Sales Rep Workflow:**
```
Step 1: Sales rep melakukan visit
Step 2: Setelah meeting, langsung ingin buat report
Step 3: Berpikir: "Project apa ini?" (project-first mental model)
Step 4: BLOCKED jika project/company belum ada â†’ Harus abandon report
```

**Masalah:**
- âŒ Sales rep harus meninggalkan report form untuk create dependencies
- âŒ Tidak sesuai mental model ("Project duluan, company adalah property dari project")
- âŒ Melanggar goal "< 5 menit per report" (jika company baru, +3-5 menit)
- âŒ Bad first-time experience (new customer visit = BLOCKED)
- âŒ Budi akan frustasi dan kembali ke WhatsApp

---

## âœ… Solution: Nested Inline Creation (Option 2)

### Konsep Solusi

**Prinsip:** User TIDAK PERNAH meninggalkan report screen, no matter what entities perlu dibuat.

**Key Features:**
- âœ… Create project inline (di dalam report form)
- âœ… Create company inline (nested di dalam project form) - **jika diperlukan**
- âœ… Create contact inline (nested di dalam project form) - **jika diperlukan**
- âœ… Select existing entities dari dropdown (path tercepat - 60% kasus)
- âœ… Flexible: Mix create new + select existing dalam satu flow

**Pattern:** "Bottom of Dropdown" - Opsi "[+ Buat [Entity] Baru]" selalu di bottom dropdown

**Kenapa Option 2 (bukan Option 1: Basic Inline):**
- Option 1 hanya allow create project inline, company/contact harus sudah ada â†’ Masih block user
- Option 2 allow create SEMUA entities inline â†’ Zero friction, handles semua scenarios
